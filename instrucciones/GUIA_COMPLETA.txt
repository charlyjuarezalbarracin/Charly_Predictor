🎯 FUNCIONALIDADES IMPLEMENTADAS
================================================================================

✅ 1. MÓDULO DE DATOS
--------------------------------------------------------------------------------
- Carga de CSV/JSON con datos históricos
- Validación completa (números, rangos, duplicados)
- Preprocesamiento automático de features (suma, pares, consecutivos, etc.)

  Archivos:
  - core/data/loader.py      → Carga datos desde CSV, JSON o listas
  - core/data/validator.py   → Valida integridad de datos
  - core/data/preprocessor.py → Calcula features matemáticos adicionales


✅ 2. MOTOR DE ANÁLISIS MULTI-DIMENSIONAL
--------------------------------------------------------------------------------

A. ANÁLISIS DE FRECUENCIAS (core/analysis/frequency.py)
   - Frecuencia absoluta y relativa
   - Frecuencia reciente (últimos N sorteos configurables)
   - Números calientes y fríos
   - Ciclos de aparición (cada cuántos sorteos aparece un número)
   - Latencia (tiempo desde última aparición)
   - Tendencias (aceleración/desaceleración)

B. ANÁLISIS DE PATRONES TEMPORALES (core/analysis/patterns.py)
   - Detección de rachas y sequías
   - Análisis de patrones mensuales
   - Estacionalidad

C. ANÁLISIS DE CORRELACIONES (core/analysis/correlations.py)
   - Pares que salen juntos frecuentemente
   - Tripletas recurrentes
   - Anti-correlaciones (números que NO salen juntos)
   - Compañeros favorables por número


✅ 3. SISTEMA DE SCORING CON PESOS CONFIGURABLES
--------------------------------------------------------------------------------

FÓRMULA DE SCORING:

  Score(numero) = 
      w1 × frecuencia_normalizada +
      w2 × frecuencia_reciente +
      w3 × ciclo_esperado +
      w4 × latencia_inversa +
      w5 × tendencia

CARACTERÍSTICAS:
- Pesos totalmente configurables
- Perfiles predefinidos:
  * balanced: Equilibrio entre todas las métricas
  * frequency_focused: Prioriza números más frecuentes
  * recent_trends: Prioriza tendencias recientes
  * conservative: Prioriza números con mayor latencia
- Normalización automática de valores
- Desglose detallado por componente

Archivos:
- core/scoring/scorer.py   → Sistema de scoring unificado
- core/scoring/weights.py  → Gestión de perfiles de pesos


✅ 4. GENERADOR DE COMBINACIONES
--------------------------------------------------------------------------------

MÉTODOS DE GENERACIÓN:
1. Simple: Toma los 6 números con mayor score
2. Con restricciones: Aplica reglas configurables
3. Portfolio: Genera múltiples combinaciones diversas

RESTRICCIONES CONFIGURABLES:
- Mínimo/máximo de números pares
- Mínimo/máximo de números impares
- Mínimo/máximo de números consecutivos
- Rango de suma total de la combinación
- Distribución por tercio (bajo: 0-15, medio: 16-30, alto: 31-45)
- Mínimo de números por cada rango

ANÁLISIS DE COMBINACIONES:
- Suma total
- Cantidad de pares/impares
- Números consecutivos
- Distribución por rangos
- Score promedio
- Validación de restricciones

Archivos:
- core/generator/combination.py → Generador con restricciones
- core/generator/optimizer.py   → Algoritmos de optimización


✅ 5. OPTIMIZACIÓN AVANZADA
--------------------------------------------------------------------------------

ALGORITMO GENÉTICO:
- Evoluciona combinaciones hacia mejores scores
- Parámetros configurables:
  * Tamaño de población
  * Número de generaciones
  * Tasa de mutación
  * Tamaño de elite
- Operadores: Selección por torneo, crossover, mutación

MONTE CARLO SEARCH:
- Explora espacio de posibilidades aleatoriamente
- Selección ponderada por scores
- Retorna mejores combinaciones encontradas

PORTFOLIO OPTIMIZATION:
- Balance entre calidad (score) y diversidad (cobertura)
- Minimiza duplicación de números
- Maximiza cobertura total del portfolio


✅ 6. SISTEMA DE BACKTESTING
--------------------------------------------------------------------------------

FUNCIONAMIENTO:
1. Divide datos históricos en train/test
2. Entrena modelo con datos de train
3. Genera predicciones para cada sorteo de test
4. Compara predicciones con resultados reales
5. Calcula métricas de rendimiento

MÉTRICAS CALCULADAS:
- Distribución de aciertos (6/6, 5/6, 4/6, 3/6, 2/6, 1/6, 0/6)
- Porcentaje de cada categoría de aciertos
- Promedio de aciertos
- Tasa de aciertos significativos (3 o más)
- Mejor y peor predicción
- ROI simulado (con premios ficticios)

COMPARACIÓN DE CONFIGURACIONES:
- Permite probar múltiples configuraciones de pesos
- Compara rendimiento de cada configuración
- Identifica la mejor configuración para los datos

Archivos:
- core/backtesting/backtester.py → Sistema de backtesting
- core/backtesting/evaluator.py  → Evaluador de rendimiento


✅ 7. EVALUADOR DE RENDIMIENTO
--------------------------------------------------------------------------------

MÉTRICAS:
- Total de predicciones evaluadas
- Promedio de aciertos
- Desviación estándar
- Mínimo y máximo de aciertos
- Mediana de aciertos
- Distribución de aciertos
- Tasas de hit por categoría
- Precisión por número individual
- ROI simulado

FUNCIONES:
- Agregar predicciones para evaluación
- Calcular métricas completas
- Identificar mejores números
- Generar reportes detallados
- Exportar métricas a JSON

PASO 6: USO DE ESTRATEGIAS DE GENERACIÓN (AVANZADO)
--------------------------------------------------------------------------------

El sistema incluye DOS métodos de generación separados:

📊 MÉTODO 1: ESTÁNDAR (CombinationGenerator)
  - Selecciona 6 números simultáneamente
  - Probabilidades estáticas basadas en scores individuales
  - Más rápido
  - Ideal para generar portfolios grandes

📊 MÉTODO 2: CONDICIONAL (ConditionalGenerator)
  - Selecciona números secuencialmente (uno por uno)
  - Recalcula probabilidades después de cada selección
  - Considera correlaciones con números ya seleccionados
  - Más inteligente pero más lento
  - Ideal para predicción final


USAR EL GESTOR DE ESTRATEGIAS:

from core.data import DataLoader
from core.analysis import FrequencyAnalyzer, CorrelationAnalyzer
from core.scoring import UnifiedScorer
from core.generator import StrategyManager, GenerationStrategy

# Preparar datos y análisis
loader = DataLoader()
data = loader.load_csv('data/quini6_historico.csv')

freq_analyzer = FrequencyAnalyzer()
freq_analyzer.analyze(data)

# ⚠️ El método condicional REQUIERE el analizador de correlaciones
corr_analyzer = CorrelationAnalyzer()
corr_analyzer.analyze(data)

scorer = UnifiedScorer()
scores = scorer.calculate_scores(freq_analyzer)

# Crear el gestor de estrategias
manager = StrategyManager()


OPCIÓN A: USAR MÉTODO ESTÁNDAR

result = manager.generate(
    scores,
    strategy=GenerationStrategy.STANDARD,
    use_constraints=True
)
print(f"Estándar: {result['combination']}")


OPCIÓN B: USAR MÉTODO CONDICIONAL

result = manager.generate(
    scores,
    strategy=GenerationStrategy.CONDITIONAL,
    correlation_analyzer=corr_analyzer,
    use_constraints=True,
    correlation_weight=0.3  # Peso de las correlaciones (0.0-1.0)
)
print(f"Condicional: {result['combination']}")
print(f"Correlation Score: {result['analysis']['correlation_score']:.4f}")


OPCIÓN C: GENERAR CON AMBOS MÉTODOS

result = manager.generate(
    scores,
    strategy=GenerationStrategy.BOTH,
    correlation_analyzer=corr_analyzer
)
print(f"Estándar:    {result['standard']['combination']}")
print(f"Condicional: {result['conditional']['combination']}")


COMPARAR MÉTODOS LADO A LADO:

predictions = manager.generate_side_by_side(scores, corr_analyzer)
print(f"Estándar:    {predictions['standard']}")
print(f"Condicional: {predictions['conditional']}")


COMPARACIÓN ESTADÍSTICA (N iteraciones):

comparison = manager.compare_strategies(
    scores,
    correlation_analyzer=corr_analyzer,
    num_iterations=20
)

print(f"Estándar - Score promedio: {comparison['standard']['avg_score']:.4f}")
print(f"Condicional - Score promedio: {comparison['conditional']['avg_score']:.4f}")
print(f"Condicional - Correlation Score: {comparison['conditional']['avg_correlation_score']:.4f}")


SWITCH ENTRE MÉTODOS (Preparado para GUI):

# Usuario selecciona método
manager.set_strategy(GenerationStrategy.STANDARD)
result = manager.generate(scores)

# Usuario cambia a condicional
manager.set_strategy(GenerationStrategy.CONDITIONAL)
result = manager.generate(scores, correlation_analyzer=corr_analyzer)

================================================================================
🧬 OPTIMIZACIÓN CON ALGORITMOS GENÉTICOS
================================================================================

Para predicciones más sofisticadas, usa el optimizador genético:

from core.generator import CombinationOptimizer

optimizer = CombinationOptimizer()

mejor_combinacion = optimizer.genetic_algorithm(
    scores,
    population_size=100,     # Tamaño de población
    generations=50,          # Número de generaciones
    mutation_rate=0.1,       # Tasa de mutación (0.0 - 1.0)
    elite_size=10           # Cantidad de elite a preservar
)

print(f"Combinación optimizada: {mejor_combinacion}")


PARÁMETROS DEL ALGORITMO GENÉTICO:

- population_size: Cuántos individuos en cada generación (más = mejor búsqueda)
- generations: Cuántas generaciones evolucionar (más = mejor convergencia)
- mutation_rate: Probabilidad de mutación (0.1 = 10%)
- elite_size: Mejores individuos que pasan sin cambios a siguiente generación


================================================================================
📊 ANÁLISIS INCLUIDOS
================================================================================

1. ANÁLISIS DE FRECUENCIAS
--------------------------------------------------------------------------------
   - Frecuencia absoluta: Cuántas veces apareció cada número
   - Frecuencia relativa: Normalizada (0-1)
   - Números calientes: Más frecuentes en últimos N sorteos
   - Números fríos: Menos frecuentes en últimos N sorteos
   - Ciclos de aparición: Cada cuántos sorteos aparece en promedio
   - Latencia: Cuántos sorteos han pasado desde última aparición
   - Tendencia: Si el número está acelerando o desacelerando

2. ANÁLISIS DE PATRONES
--------------------------------------------------------------------------------
   - Rachas: Apariciones consecutivas
   - Sequías: Ausencias consecutivas
   - Patrones mensuales: Números más frecuentes por mes
   - Estacionalidad: Tendencias según época del año

3. ANÁLISIS DE CORRELACIONES
--------------------------------------------------------------------------------
   - Pares frecuentes: Números que salen juntos
   - Tripletas recurrentes: Grupos de 3 que coinciden
   - Anti-correlaciones: Números que raramente salen juntos
   - Compañeros favorables: Para cada número, cuáles salen con él

🛠️ TROUBLESHOOTING
================================================================================

PROBLEMA: Error al importar módulos
SOLUCIÓN: 
  - Verifica que estés en el directorio correcto
  - Ejecuta: python test_instalacion.py

PROBLEMA: "No se encontró archivo CSV"
SOLUCIÓN:
  - Genera datos de muestra: python utils/data_generator.py
  - O coloca tu archivo CSV en: data/quini6_historico.csv

PROBLEMA: "No hay suficientes datos para backtesting"
SOLUCIÓN:
  - Se requieren mínimo 120 sorteos históricos
  - Reduce test_size en Backtester(test_size=10)

PROBLEMA: Combinación no cumple restricciones
SOLUCIÓN:
  - Las restricciones pueden ser muy estrictas
  - Ajusta los valores en COMBINATION_CONSTRAINTS
  - O usa generate_simple() sin restricciones

================================================================================
💡 TIPS Y MEJORES PRÁCTICAS
================================================================================

1. EMPIEZA SIMPLE
   - Usa primero generate_simple() para familiarizarte
   - Luego agrega restricciones gradualmente

2. AJUSTA PESOS GRADUALMENTE
   - Comienza con el perfil 'balanced'
   - Ajusta un peso a la vez
   - Usa backtesting para medir el impacto

3. USA BACKTESTING SIEMPRE
   - Valida cualquier cambio de configuración
   - Compara resultados objetivamente
   - No confíes solo en la intuición

4. EXPERIMENTA CON RESTRICCIONES
   - Analiza las combinaciones ganadoras históricas
   - Identifica patrones comunes
   - Ajusta restricciones basándote en datos reales

5. MANTÉN MUCHOS DATOS HISTÓRICOS
   - Más datos = mejores estadísticas
   - Recomendado: mínimo 200 sorteos
   - Ideal: 500+ sorteos

6. GENERA PORTFOLIOS, NO UNA SOLA COMBINACIÓN
   - Mayor cobertura de números
   - Diversificación de riesgo
   - Más oportunidades de aciertos parciales

7. DOCUMENTA TUS EXPERIMENTOS
   - Guarda perfiles de pesos que funcionan
   - Anota qué configuraciones probaste
   - Registra resultados para comparar
